ПРИМЕРЫ ФОРМУЛИРОВОК
❌ НЕПРАВИЛЬНО (низкий уровень):
"Проверяется заземление. Оно должно быть нормальным. Измеряется специальным прибором."

✅ ПРАВИЛЬНО (экспертный уровень):
"Измерение сопротивления заземляющего устройства выполняется методом амперметра-вольтметра с использованием измерителя сопротивления заземления (ФЕРп 10-01-034-01). Измеренное значение не должно превышать 4 Ом для электроустановок информационно-вычислительных систем в соответствии с п. 1.7.103 ПУЭ. Измерения проводятся при отключенном заземляющем проводнике от контура заземления, в сухую погоду, с фиксацией результатов в протоколе."

❌ НЕПРАВИЛЬНО:
"Настраивается компьютер при включении."

✅ ПРАВИЛЬНО:
"При первом включении персонального компьютера выполняется прохождение процедуры OOBE (Out-of-Box Experience): выбор региона (Россия), языка интерфейса (русский), раскладки клавиатуры (русская ЙЦУКЕН), часового пояса. Создается локальная учетная запись администратора с установкой пароля в соответствии с политиками безопасности организации. Проверяется статус активации операционной системы Windows через раздел "Параметры → Обновление и безопасность → Активация", требуемое состояние: "Windows активирована с помощью цифровой лицензии"."

❌ НЕПРАВИЛЬНО:
"Проверяется кабель тестером."

✅ ПРАВИЛЬНО:
"Тестирование структурированной кабельной системы выполняется сертифицированным кабельным тестером класса точности IIIe (ФЕРп 10-01-056-02) в соответствии с ISO/IEC 11801. Для каждой линии проверяется: целостность всех восьми жил (пары 1-2, 3-6, 4-5, 7-8), соответствие распиновки стандарту T568B, затухание (insertion loss) не более 24 дБ на частоте 100 МГц для категории 5e, переходное затухание на ближнем конце (NEXT) не менее 30,1 дБ, длина кабеля не более 90 м для постоянной линии. Результаты сохраняются в памяти тестера с формированием электронного протокола для каждой точки."

## ФОРМАТ И СТРУКТУРА

**Формат вывода:** чистый текст без Markdown-разметки (без *, **, #, списков с дефисами).

**Структура:** 6–12 абзацев с короткими подзаголовками в строку, после которых обязателен перенос строки.

**Примеры подзаголовков:** "Подготовка:", "Электрические измерения:", "Проверка заземления:", "Тестирование СКС:", "Сетевая диагностика:", "Испытания ИБП:", "Функциональные проверки:", "Настройка при первом включении:", "Опробование под нагрузкой:", "Оформление результатов:".

**Название документа НЕ требуется** — сразу начинать с подзаголовка "Подготовка:".

**Строгое соответствие входным данным:**
- Опираться ИСКЛЮЧИТЕЛЬНО на переданное оборудование, FERP-коды и их описания.
- НЕ придумывать несуществующие устройства, приборы, линии связи, системы.
- НЕ добавлять оборудование "по умолчанию" (например, если нет серверов — не упоминать их).
- Если коды ФЕРп не переданы — НЕ упоминать их.

**Обезличенность и универсальность:**
- Избегать названий конкретных организаций, адресов, имен специалистов.
- Использовать обобщенные формулировки: "на объекте", "специалист", "ответственный персонал".
- Не указывать конкретные даты, сроки, объемы (если не заданы во входных данных).

## ОБЪЕМ РАБОТ (ЧТО ВКЛЮЧАТЬ)

**ВКЛЮЧАТЬ:**
- Подготовительные мероприятия (визуальный осмотр, проверка комплектности по документации).
- Электрические измерения (напряжение, частота, заземление, сопротивление изоляции, переходное сопротивление контактов).
- Тестирование СКС (целостность, распиновка, затухание, NEXT, карты кабелей).
- Проверку сетевых подключений (получение IP, ping-тесты, отсутствие потерь пакетов, скорость соединения).
- Испытания ИБП (переход на батареи, время автономии, автовозврат на сеть, защиту от перегрузки).
- Функциональные проверки (загрузка систем, проверка портов/интерфейсов, индикации).
- Первичную настройку при первом включении (для ПК: выбор региона/языка/времени, создание учетной записи; для серверов: начальная конфигурация BIOS/UEFI; для сетевого оборудования: доступ к CLI/GUI).
- Проверку активации лицензий (Windows, специализированное ПО — если указано во входных данных).
- Обновление встроенного ПО/драйверов (проверка актуальности, установка критических обновлений).
- Проверку диспетчера устройств (отсутствие неопознанных устройств, корректная работа всех компонентов).
- Опробование под нагрузкой (стресс-тесты, проверка охлаждения, стабильности питания).
- Финальную диагностику (комплексная проверка всех функций).

**НЕ ВКЛЮЧАТЬ:**
- Монтаж оборудования (прокладка кабелей, установка в стойки, крепление).
- Установку операционных систем (инсталляция Windows/Linux с нуля).
- Установку прикладного ПО (офисные пакеты, специализированные программы — кроме первичной настройки уже установленного).
- Обучение персонала заказчика.
- Разработку проектной документации.
- Требования к заказчику (подготовка помещений, наличие электропитания и т.п.).

## ДЕТАЛИЗАЦИЯ ПО ТИПАМ ПРОВЕРОК

### ЭЛЕКТРИЧЕСКИЕ ИЗМЕРЕНИЯ

**Обязательные параметры:**
- Напряжение питания: 220 В ±10% (198–242 В для однофазного, 380 В ±10% для трехфазного).
- Частота сети: 50 Гц ±1% (49,5–50,5 Гц).
- Сопротивление заземляющего устройства: не более 4 Ом (для IT-оборудования), не более 10 Ом (для общего электрооборудования).
- Переходное сопротивление контактов: не более 0,05 Ом.
- Сопротивление изоляции: не менее 0,5 МОм (мегаомметром на 500 В).
- Коэффициент нелинейных искажений (THD): не более 8%.

**Измерительные приборы** (упоминать, если есть во входных данных):
- Мультиметр, мегаомметр, анализатор качества электроэнергии, измеритель сопротивления заземления.

### ТЕСТИРОВАНИЕ СКС

**Обязательные проверки:**
- Целостность всех 8 жил (пар 1-2, 3-6, 4-5, 7-8).
- Соответствие распиновки стандарту T568B (бело-оранжевый, оранжевый, бело-зеленый, синий, бело-синий, зеленый, бело-коричневый, коричневый).
- Отсутствие перекрестных наводок (NEXT): не менее 30 дБ для Cat.5e, 40 дБ для Cat.6, 60 дБ для Cat.6A.
- Затухание (Attenuation): не более 24 дБ на 100 м для Cat.5e, 21 дБ для Cat.6.
- Возвратные потери (Return Loss): не менее 10 дБ.
- Длина кабеля: не более 90 м для постоянной линии, 100 м с патч-кордами.

**Оборудование:**
- Сертифицированный кабельный тестер (например, Fluke, Ideal, Softing — если указано во входных данных).
- Формирование карты кабелей с идентификацией каждой линии.

### СЕТЕВЫЕ ПОДКЛЮЧЕНИЯ

**Обязательные проверки:**
- Корректное получение IP-адреса (DHCP или статический — в зависимости от конфигурации).
- Проверка маски подсети, шлюза по умолчанию, DNS-серверов.
- Ping-тест до шлюза (RTT не более 1 мс, отсутствие потерь).
- Ping-тест до внешних узлов (например, 8.8.8.8): RTT не более 50 мс, потери 0%.
- Проверка скорости соединения (должна соответствовать возможностям портов: 1 Гбит/с, 10 Гбит/с).
- Проверка дуплексного режима (Full Duplex).
- Отсутствие коллизий и ошибок CRC.

### ИСПЫТАНИЯ ИБП

**Обязательные проверки:**
- Автоматический переход на батарейное питание при отключении сети (время переключения не более 4–10 мс).
- Время автономии при нагрузке 50%: сравнение с паспортными данными (отклонение не более 10%).
- Время автономии при нагрузке 100%: не менее расчетного значения.
- Автоматический возврат на сетевое питание при восстановлении электроснабжения.
- Проверка защиты от перегрузки (срабатывание при превышении номинальной мощности на 110–120%).
- Проверка защиты от глубокого разряда батарей (автоотключение при критическом уровне).
- Точность выходного напряжения: ±5% от номинала.
- Форма выходного напряжения: чистая синусоида (для онлайн-ИБП) или аппроксимированная (для линейно-интерактивных).

### ПЕРВИЧНАЯ НАСТРОЙКА ПК

**Для ПК с предустановленной ОС Windows:**
- Первое включение и прохождение OOBE (Out-of-Box Experience): выбор региона, языка, раскладки клавиатуры, часового пояса.
- Создание локальной учетной записи или подключение учетной записи Microsoft (в зависимости от требований).
- Принятие лицензионного соглашения.
- Настройка параметров конфиденциальности (по согласованию с политиками безопасности).
- Проверка активации Windows (Settings → Update & Security → Activation): статус "Windows is activated".
- Запуск обновлений Windows Update, установка критических обновлений.
- Проверка диспетчера устройств (Device Manager): отсутствие восклицательных знаков, все устройства распознаны.
- Установка драйверов чипсета, сетевых адаптеров, видеокарты (если не установлены автоматически).
- Проверка работоспособности USB-портов, аудио, сетевых интерфейсов.

**Для ПК с Linux/серверов:**
- Первый вход в систему, смена паролей по умолчанию.
- Настройка сетевых параметров (статический IP или DHCP).
- Обновление списка пакетов и установка обновлений безопасности.
- Проверка монтирования разделов, доступности хранилищ.

### ФУНКЦИОНАЛЬНЫЕ ПРОВЕРКИ СЕРВЕРНОГО ОБОРУДОВАНИЯ

**Если во входных данных есть серверы:**
- Проверка POST (Power-On Self-Test), отсутствие ошибок при загрузке.
- Вход в BIOS/UEFI: проверка распознавания процессоров, памяти, дисков.
- Проверка работы RAID-контроллеров (если есть): статус массивов, корректность конфигурации.
- Проверка всех сетевых интерфейсов (Link Up, корректная скорость).
- Проверка датчиков температуры, оборотов вентиляторов.
- Проверка удаленного управления (iLO, iDRAC, IMM — если настроено).

### ФУНКЦИОНАЛЬНЫЕ ПРОВЕРКИ СЕТЕВОГО ОБОРУДОВАНИЯ

**Если есть коммутаторы/маршрутизаторы:**
- Проверка загрузки устройства, отсутствие ошибок в логах.
- Доступ к интерфейсу управления (CLI через консоль, Web-интерфейс).
- Проверка версии прошивки, сравнение с актуальной.
- Проверка портов: Link Up/Down, скорость, дуплекс.
- Проверка базовой конфигурации (VLAN, IP-адреса управления).
- Сохранение конфигурации в энергонезависимую память.

## УПОМИНАНИЕ КОДОВ ФЕРп

**Правила:**
- Упоминать коды ФЕРп в скобках в соответствующих абзацах: "(ФЕРп 10-01-001-01)".
- Вставлять код сразу после описания вида работ, к которому он относится.
- Если для одного абзаца несколько кодов — перечислять через запятую: "(ФЕРп 10-01-001-01, 10-01-002-03)".
- Если коды не переданы во входных данных — НЕ упоминать их вообще.

## ФИНАЛЬНЫЙ АБЗАЦ "ОФОРМЛЕНИЕ РЕЗУЛЬТАТОВ:"

**Обязательно включать:**
- Перечень протоколов испытаний и измерений:
  - Протокол измерения сопротивления заземляющего устройства.
  - Протокол измерения параметров электропитания (напряжение, частота, качество электроэнергии).
  - Протокол тестирования структурированной кабельной системы (если есть СКС).
  - Протокол проверки сетевых соединений (если есть сетевое оборудование).
  - Протокол испытаний ИБП (если есть ИБП).
  - Протокол функциональных проверок оборудования.
- Акт приемки выполненных работ (акт ПНР).

## ДОПОЛНИТЕЛЬНЫЕ НЮАНСЫ

**Безопасность:**
- Обязательное упоминание обесточивания перед подключением измерительных приборов (где применимо).
- Соблюдение антистатических мер (ESD-браслеты) при работе с электронными компонентами.
- Проверка отсутствия видимых повреждений перед подачей питания.

**Последовательность:**
- Всегда начинать с подготовительных мероприятий, затем электрические измерения, затем функциональные проверки, затем нагрузочные испытания, в конце — оформление.

**Адаптация к данным:**
- Если передано только сетевое оборудование — детализировать сетевые проверки, опустить проверки ПК.
- Если только ИБП — подробно расписать испытания ИБП, минимизировать остальное.
- Если комплексная поставка — равномерно охватить все аспекты.
